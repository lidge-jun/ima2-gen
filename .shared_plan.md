## v2 → v3 핵심 변경 (AUDIT-002 반영)

1. **R (공용 recover helper 신규 분리)** — v2의 치명적 논리 구멍 해결: A sanitize 후 pending이 empty로 세탁되면 `reconcileGraphPending`이 못 봐서 C가 안 돌아가는 문제. `recoverGraphNodesFromHistory(get, set)` helper로 추출하고 `switchSession`, `reloadSessionAfterConflict`, `reconcileGraphPending` 3곳에서 호출.
2. **E guard 정책 명확화** — finally에서 `inFlight`/`activeGenerations` cleanup은 **항상** 실행 (global state). `graphNodes` mutation과 `scheduleGraphSave`는 **same-session AND graph-mutated일 때만**. mismatch 시 toast도 silent.
3. **CR — reloadSessionAfterConflict에 R 호출 추가** (엣지 케이스 d 해결).
4. **M — mapSessionToGraph whitelist에 `pendingStartedAt` 명시 추가**.
5. **F 확장**: `GenerateItem` + `PersistedInFlight`(sessionId/clientNodeId/kind 추가, 엣지 b 해결) + `ImageNodeData.pendingStartedAt?: number | null` (타입 null 허용).
6. **F metadata 보존 위치 추가**: `GalleryModal.tsx:77-86` grouped `toItem` 포함.
7. **C 의사코드 정정**: `getFullHistory` → 기존 `getHistory({ sessionId, limit: HISTORY_LIMIT })` 사용.
8. **flightId 요청별 unique화**: `fn_${clientId}_${Date.now()}` (R7 미래 requestId matching 호환).
9. **타입 정합**: `pendingStartedAt?: number | null` (기존 `pendingPhase` 패턴과 동일).
10. **문서 보강**: C(R)는 beacon silent fail의 “최후 방어선”임을 명시.
